# KB-002: Printer Offline Troubleshooting

## Use When

A user reports a network printer as offline, jobs stuck in queue, repeated print failures, or print output going to the wrong device.

## Safety Notes

- Do not install unapproved drivers from vendor download pages without technician approval.
- Do not change shared print queues without checking whether other users are affected.
- Escalate repeated department-wide failures or printer errors that indicate hardware service is needed.

## Tier 1 Checklist

1. Confirm the printer name, location, and whether other users are affected.
2. Ask the user to verify the printer is powered on, connected, and showing no paper or toner alerts.
3. Confirm the user is connected to the correct network or VPN if printing remotely.
4. Ask the user to print a test page from another application.
5. Check for stuck jobs and restart the local print queue if approved.
6. Confirm whether the printer is set as default or selected correctly in the application.
7. Escalate if the device is unreachable, reports a hardware fault, or multiple users are impacted.

## User Response Template

Hi {{user_first_name}}, I can help with the printer issue. Please confirm the printer name or location, and whether anyone else nearby is seeing the same offline message.

## Escalation Template

User reports printer offline. Tier 1 checks completed: printer/location confirmed, network state reviewed, queue symptoms gathered, user impact checked. Please review print server queue, printer network reachability, driver status, and hardware alerts.
