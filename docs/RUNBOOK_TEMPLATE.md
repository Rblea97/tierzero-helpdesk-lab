# TierZero Runbook Template

Use this template for public, fictional knowledge-base articles and runbooks. Do not include real internal procedures, credentials, tenant names, server names, or customer data.

## Title

`KB-###: Short Issue Name`

## Use When

Describe the ticket symptoms that should match this article.

## Safety Notes

- List actions that require identity verification.
- List actions that require technician approval.
- List sensitive information the user should never send.
- List escalation triggers.

## Tier 1 Checklist

1. First safe diagnostic step.
2. Second safe diagnostic step.
3. Third safe diagnostic step.
4. Escalation decision.

## User Response Template

Hi {{user_first_name}}, short user-facing response with no sensitive instructions.

## Internal Notes Template

Summarize what was checked, what was observed, and what remains unknown.

## Escalation Template

Summarize issue, scope, completed Tier 1 steps, affected user/device, suspected cause, and requested Tier 2 review.
